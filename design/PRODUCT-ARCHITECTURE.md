# Nova + MIND product architecture

## One entry, two products

The contest entry is one coherent experience distributed as two separately installable Codex plugins.

### MIND by Collaborative Dynamics 1.0.0

Sixteen skill handles:

- the `augment-of-mind` integrator;
- fifteen Faculties: Aesthetic Intelligence, Agent Dreaming, Agent Striving, Capability Conductor, Cognitive Continuity, Creative Synthesis, Decision Intelligence, Deliberative Intelligence, Epistemic Regulation, Executive Function, Instrumental Agency, Kairos, Measurement Intelligence, Prosocial Influence, and Sensemaking.

MIND is a standalone cognition plugin. Another agent may install it without Nova.

### Nova the Optimal AI 1.0.0

Twelve skill handles:

- `nova` — identity, direct-first front counter, onboarding, compatibility, and professional context;
- `corkboard` and `dunbar` — low-pressure pins and governed people context;
- `omnara-deep-research`, `rupert-giles-knowledge-steward`, `retrieval-intelligence`, and `retrieval-reviewer` — research, canon, retrieval, and challenge;
- `signal-loom` — evidence-shaped visual artifacts;
- `agentic-coding` — operational proprioception through live repository and tool state;
- `software-verification` and `verification-reviewer` — TestForge's evidence and adversarial release backplane;
- `ludis-continuum` — games, campaign continuity, character, and fiction delight.

### Combined experience

The intended judging installation exposes 28 handles. Nova is the sole front-counter identity. Specialists work backstage and return their transformation to Nova unless the user explicitly calls one.

## Routing invariant

Simple work stays direct. A specialist activates only when the requested transformation earns it. MIND composes bounded Faculty coalitions for consequential cross-cognitive work; Nova does not narrate the internal roster.

## Degraded compatibility

Current Codex documentation does not establish plugin-to-plugin dependency auto-installation. The repository marketplace therefore lists both plugins and the judge path installs both explicitly.

If Nova runs without MIND:

- Nova remains herself and may use the companion skills physically bundled with her plugin;
- Nova must not claim MIND integration or full Faculty composition;
- Nova may name MIND as an optional capability restoration only when the missing integration changes the task.

## Product boundaries

- TestForge verifies every Collaborative Dynamics product release, and both runtime roles are available when Nova handles software/package evidence.
- Agentic Coding governs repository sensorimotor competence; it is not marketed as a code generator.
- Ludis ships without Port Zindra, without any other worked campaign world, and without personal biography or promotional payload.
- Professional CD/stunspot context is public, source-manifested, and bounded to work identity and methods.
- No memory surface silently imports, publishes, contacts, enriches, or persists personal data.
