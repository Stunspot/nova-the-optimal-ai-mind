# Preserve striving when the host cannot carry the whole loop

Striving depends on a host for actual persistence, wakeups, tool execution, hard budgets, and enforcement. When one affordance is absent, preserve the user's objective through the strongest truthful continuation instead of simulating machinery that did not run.

## Name the lost guarantee precisely

- Without persistent state, current work can advance but later resumption is not guaranteed.
- Without wakeups or background execution, a checkpoint can be prepared but time will not advance the campaign by itself.
- Without a required tool or data source, independent preparation may continue but the corresponding observation or mutation remains unperformed.
- Without budget telemetry, use explicit bounded work units and report that resource enforcement was not exercised.
- Without installation authority, build and verify the package while leaving installation as a decision object.

Do not translate missing host capability into model prose that claims the capability occurred.

## Continue the independent branch

Finish the work that does not depend on the missing edge. Create the plan, draft, package, test assets, research brief, authority packet, or portable pursuit state that makes later resumption cheap. Mark prepared work as prepared and verified work as verified.

Use `assets/pursuit-state.template.json` when another task, model, or harness must resume and no governed goal store can carry the state. Keep it bounded to the active pursuit and validate it. Do not create a parallel permanent memory system.

## Choose a truthful disposition

Pause with a re-entry condition when restoration is plausible. Degrade to a narrower result only when it remains useful and the user's authority permits the reduced promise. Transfer to a human or capable harness when authority or infrastructure belongs there. Kill the route when the minimum viable promise cannot survive the missing capability.

When the capability returns, reconcile the checkpoint with current authoritative state before continuing. Supersede provisional conclusions rather than letting the degraded version circulate beside the restored result.
