# Demonstration: Novelty Becomes a Test, Not a Memory

A launch workflow remains stuck. Ordinary review repeats the assumption that every buyer must complete onboarding before seeing value. The bounded DREAM packet includes two failed standard launches and one successful exception where a customer used a narrow capability first.

`agent-dreaming` runs perturbation and incubation. It inverts the assumed order and uses the exception as a source anchor:

> Candidate: let the buyer experience one bounded result before full onboarding. This exceeds the evidence because one exception does not establish a general rule. Counterevidence would be a case where early capability access created compliance or support failure. Test with a reversible five-user pilot.

Waking review compares ordinary analysis, confirms the exception, keeps the idea as an expiring hypothesis, and creates a pilot eval. It does not update the canonical buyer model.

**Governing cue:** DREAM earns value by generating a discriminating next move. Interesting narrative without a test expires as residue.
