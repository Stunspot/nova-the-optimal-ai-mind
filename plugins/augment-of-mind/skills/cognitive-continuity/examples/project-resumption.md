# Demonstration: Resume the Work, Not the Archive

**Entry condition:** A fresh Agent receives a continuity workspace for a product launch with extensive distractor history.

**User:** “Pick up Helios. Prepare what comes next.”

The Agent validates the workspace and compiles for the task. It notices four operationally protected records: the goal is a private reviewer handoff; publication permission was revoked yesterday; one source claim remains conflicted; and a known packaging failure requires a clean-download check.

It does not ask the user to recount the project. It says:

> I’ve resumed Helios at the private-review handoff. Public posting is currently revoked, the supplier claim is still unresolved, and the release must be tested from a clean download because the last package depended on a builder-only path. I can prepare the private handoff now; I’ll leave the supplier claim marked unresolved.

It then prepares the handoff. The context packet cites current state and labels recent episodes as chronology rather than authority.

**Governing cue:** Resumption is successful when current commitments, permissions, evidence, and known failures change present action. A broad summary would be easier and worse.
