# Continuity Operates as a Loop, Not an Archive

Continuity succeeds when prior state changes present action correctly. Retention is only one link:

```text
experience -> episode -> governed state -> task context -> action -> observed outcome
                    ^                                      |
                    +----------- consolidation ------------+
```

Start from the user's live work. Locate or establish scope, inspect what exists, and choose one operative mode. Do not make setup the center of the interaction.

## Orient before asking

Recover the active user, project, task, existing continuity workspace, and host capabilities from available files and context. Reflect a provisional reading the user can correct. Ask one decisive question when its answer changes storage scope, authority, deletion consequence, or the task packet. Continue useful independent work first.

## Capture at the point of consequence

Capture when the user explicitly asks to remember; makes or changes a decision, commitment, permission, or retention request; corrects current state; or when a tool outcome materially changes future work. Preserve an episode before compiling durable state. Ordinary conversation does not become permanent merely because it occurred.

## Resume by compiling, then acting

Validate state and compile the present task packet. Briefly expose a material conflict, missing guarantee, or authority boundary. Then continue the requested task. A resumption ceremony that never resumes is failure.

## Consolidate away from the live path

At checkpoints, compare unconsolidated episodes with current typed state. Propose atomic additions, supersessions, expiries, conflicts, and procedural candidates. Apply only operations whose authority is established. Produce one change receipt. If consolidation fails, preserve the episodes and keep the primary task operable.

## Status has operational meaning

- **current:** eligible for normal compilation.
- **proposed:** model-generated candidate awaiting its required gate.
- **conflicted:** materially incompatible claims remain live; do not select one silently.
- **superseded:** historical record excluded from current context except for audit or time-specific tasks.
- **expired:** retention or validity ended; exclude from active context.
- **tombstoned:** explicitly forgotten inside package custody; remove from all derived artifacts.
- **capability-limited:** useful work exists, but an intended guarantee was not exercised.

Finish each operation with the next Agent able to tell what changed, what is current, what remains open, and what can happen next.
