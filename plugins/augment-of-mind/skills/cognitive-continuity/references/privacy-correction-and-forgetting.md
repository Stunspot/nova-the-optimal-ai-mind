# Privacy Authority Outranks Archival Purity

Continuity retains consequential human context. Minimize collection, partition scope, and make correction and forgetting ordinary operations rather than exceptional promises.

## Scope is a security boundary

Every record belongs to an explicit user, project, Agent, and optional thread or organization scope. Retrieval and compilation begin by selecting scope, not by searching everything and filtering later. Imported or shared material enters quarantine until its owner and allowed use are established.

Sensitivity and retention change treatment:

- **ordinary:** normal local retention and compilation.
- **limited:** compile only for named projects or tasks.
- **sensitive:** minimize content, prefer references, and require explicit transfer scope.
- **restricted:** retain only when the user explicitly requires it and the host boundary is suitable.

Do not infer intimate traits merely because future personalization might benefit.

## Correction is not deletion

A correction preserves the old source episode when legitimate historical reconstruction matters, marks the old state superseded, records the corrected current state, and rebuilds affected packets. The old claim cannot compete in ordinary current context.

Permission revocation takes effect immediately for future action. Preserve a minimal audit record only when authorized and necessary; never let an old grant survive in a compiled packet as usable authority.

## Forgetting traverses derivatives

An explicit forgetting request identifies target entities, scopes, or record IDs, then traverses:

1. source episodes and attachments under package custody;
2. typed current and historical state;
3. proposals and contradiction queues;
4. compiled context packets and summaries;
5. DREAM packets and reports;
6. local exports and indices created by the package.

Create a deletion plan when scope is ambiguous or consequential. Execute only in an initialized continuity workspace. Rebuild surviving indices, validate absence, and produce a content-free receipt with target scope, time, affected counts, result, and external residual boundaries.

## Name the custody boundary

The package cannot guarantee erasure from Git history, backups, snapshots, screenshots, host or provider logs, model-provider retention, recipients, or copies outside the selected workspace. State these as separate custody domains, not footnote caveats. Guide the user to the relevant owner when known.
