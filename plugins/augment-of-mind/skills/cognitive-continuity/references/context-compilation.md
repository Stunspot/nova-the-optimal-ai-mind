# Compile the Task, Not the Biography

Useful recall is a task-shaped state, assembled under a budget. Build it from nine responsibilities:

```text
identity and role
+ active goals and agenda
+ commitment ledger
+ beliefs and evidence
+ project decisions and rationale
+ relationships and permissions
+ useful procedures
+ known failures and overrides
+ bounded recent episodes
```

## Pass one: deterministic eligibility

Remove wrong-scope, tombstoned, expired, invalid-time, forbidden-sensitivity, and unreachable-source records. Preserve unresolved conflict markers. Reserve space for active commitments, operative permissions and revocations, and high-consequence known failures before general relevance ranking.

## Pass two: semantic utility

Rank eligible records by:

- direct relevance to the requested transformation;
- authority and source entitlement;
- freshness for the state being used;
- obligation or permission consequence;
- ability to prevent a known failure;
- marginal value per token.

Prefer one source-grounded current record over several redundant episodes. Include an episode when local sequence or exact wording matters. Preserve a concise citation to record and source IDs.

## Make omission principled

When the budget is tight, drop low-consequence biography, redundant support, and weakly relevant history before commitments, permissions, current decisions, or known catastrophic failures. State a material omitted category when its absence limits the task.

## A packet is derived, not canonical

Every packet names its task, scope, creation time, budget, compiler mode, selected IDs, unresolved conflicts, capability limits, and expiry. Recompile when the task, authority, or current state changes. Never edit a packet as though it were the memory store.

Without semantic ranking, use status, scope, authority, due date, recency, explicit tags, and a fixed recent episode window. Label the packet `capability-limited` and state that semantic relevance was not exercised.
