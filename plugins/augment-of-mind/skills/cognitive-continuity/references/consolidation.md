# Consolidation Is Conservative Model Maintenance

Consolidation converts episodes into reviewable changes without placing memory maintenance inside live task reasoning.

## Replay before abstraction

Select unconsolidated episodes by scope and checkpoint. Reconstruct what was observed, asserted, decided, permitted, attempted, and verified. Keep source IDs attached while extracting atomic candidates.

Compare each candidate with current typed state:

- **add:** no current equivalent exists and persistence is useful;
- **reinforce:** new evidence supports a current record without changing it;
- **supersede:** authority and valid time establish a new current state;
- **conflict:** incompatible live claims lack a responsible resolution;
- **expire:** validity or retention ended;
- **no durable change:** the episode matters historically but not as current state;
- **procedure candidate:** repeated outcomes suggest reusable competence.

## Down-select without deleting truth

Merge redundant expression at the state layer while preserving source episodes. Reduce weak, stale, or low-utility associations from active compilation through status, expiry, or lower salience. Use explicit forgetting, not ordinary consolidation, to remove retained source content.

## Stage authority by consequence

An explicit low-risk user instruction may support an immediate typed write with a receipt. Derived user traits, relationship claims, sensitive inferences, consequential permissions, disputed facts, identity changes, and procedures require review or their accountable owner. Equal-authority conflict never self-resolves.

## Learning has a longer chain

A successful episode does not become a SKILL. Preserve preconditions, operation, artifacts, validators, failures, repetitions, and transfer evidence. Route a recurrent gap or opportunity through `procedural-learning-handoff.md`. Pathfinder decides whether competence is missing; Builder constructs it; TestForge examines the claim; only authorized installation changes operative capability.

Finish with accepted, rejected, deferred, and conflicted proposal IDs plus the last consolidated episode. A failed proposal must not strand the event log or block ordinary work.
