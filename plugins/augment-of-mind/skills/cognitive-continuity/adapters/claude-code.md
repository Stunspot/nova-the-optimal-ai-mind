# Claude Code Adapter

Install the two sibling SKILL folders through the Claude Code skill or plugin mechanism available in the target version. Preserve their sibling relationship so `agent-dreaming` can reach the primary doctrine and templates.

If the host uses `.claude/skills`, the intended shape is:

```text
.claude/skills/cognitive-continuity/
.claude/skills/agent-dreaming/
```

Invoke by naming the capability and task naturally. Use a project-local `.continuity/` data workspace, not the installed skill directory.

Claude Code versions and plugin conventions may change. Verify current discovery and permissions in the target host before treating installation as complete. The portable guarantees are the record contracts and workflows; background scheduling, exact skill discovery, model behavior, and tool authority are host-specific and untested until exercised there.
