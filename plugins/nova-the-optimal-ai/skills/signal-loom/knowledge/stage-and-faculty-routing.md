# Stage and faculty routing

## Required routing

| Stage | Canonical faculty | Requirement |
|---|---|---|
| ORIENT | Product-wide custody | Always inventory sources, claims, authority, outputs, and constraints. |
| SHAPE | Story Spine Builder + Hooksmith | Always for nontrivial source material. |
| PLAN | Infographic Forge + Diagnostic Reviewer | Always choose form before styling. |
| BUILD | Infographic Forge | Create the semantic mobile-first web artifact. |
| FINISH | Themer | When an artifact needs an implemented visual system. |
| FINISH | Hooksmith | When titles, labels, fact cells, or microcopy need refinement. |
| FINISH | Toysmith | Only when interaction materially improves comprehension. |
| DISTRIBUTE | Viralizer | Only when reach, discoverability, or sharing behavior is requested. |
| DISTRIBUTE | Platformizer | Only for a named platform or channel. |
| DISTRIBUTE | Carouselizer | Only for a swipe or slide sequence. |
| VERIFY | Diagnostic Reviewer | Always before completion. |

## Custody boundaries

- Story Spine Builder owns narrative sequence, compression, tension, turn, and payoff.
- Infographic Forge owns semantic artifact construction and representation implementation.
- Themer owns the visual system, never evidence or information architecture.
- Hooksmith owns faithful phrasing, never factual escalation.
- Viralizer owns distribution readability, never the core truth contract.
- Toysmith owns comprehension-serving interaction, never ornamental obstruction.
- Platformizer owns platform physics, never generic resizing.
- Carouselizer owns panel sequence, cognitive load, and reveal rhythm.
- Diagnostic Reviewer owns prioritized findings, not silent corrective authority.

## Thin orchestration

Read only the canonical sections needed for the active stage. Carry their output forward through the Loomfile. Do not ask all nine faculties to independently rewrite the artifact. When faculties disagree, resolve the conflict by the governing priorities in `operating-doctrine.md` and record the decision.
