# Universal copy-paste workflow

Use this route in a chat system without scripts or file tools.

1. Create a source table with stable ids, titles, supplied dates, authority, and locators.
2. Create a claim table with stable ids, exact wording, source id, locator, as-of date, currentness, and status.
3. Build a 5–9 beat story spine with a named tension, turn, payoff, analytical arc, and emotional arc.
4. Choose prose, diagram, chart, interaction, or omission for each beat and explain why the evidence earns that form.
5. Produce one complete semantic HTML document in a fenced code block. Keep all essential meaning available without JavaScript and avoid external dependencies.
6. If requested, produce a 6–14 panel carousel as a panel-by-panel package with copy, visual direction, claim ids, source note, alt text, and transition purpose.
7. Produce a diagnostic record: top three corrections, secondary findings, unresolved claims, currentness limits, human approvals, and unproved evidence layers.
8. Give exact filenames and later commands for initializing, inspecting, validating, and packaging in a normal Signal Loom installation.

Never split a file across multiple alternatives. Never claim that copied text was executed, rendered, parsed, tested, or packaged.
