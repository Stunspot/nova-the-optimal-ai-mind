# Claude Code overlay

Install or expose both skill directories in the host's supported skill location while keeping `references`, `assets`, `scripts`, `examples`, and `evals` package-relative. Begin from the repository root and preserve the target project's own instructions and package manager.

Tool permission does not equal user authorization. Run local non-destructive inspection and narrow tests when permitted. Keep production-code and test changes inside scope already authorized; ask again only when the action crosses it. Dependency installation, CI or deployment changes, production access, active security work, destructive operations, and external actions retain separate authority edges. Use a fresh context for the independent reviewer when the host supports it.
