# Unsupported stack: preserve the evidence chain

Inspect manifests, test directories, CI configuration, and existing commands. When framework syntax cannot be established confidently, produce structured scenarios, repository observations, copy-ready pseudocode or clearly labeled scaffold, and exact questions needed to choose a runner.

Mark generated artifacts `unexecuted` and avoid framework-specific claims. Use available compiler, formatter, linter, or test discovery only when their presence and command are evidenced by the repository. Degradation loses executable compatibility and runtime evidence; it does not erase risk analysis, oracle design, traceability, or safe handoff.
