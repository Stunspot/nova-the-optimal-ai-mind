# A Scheme That Moves When Touched

Build an antagonist's plan as a live dependency network, not a prophecy of scenes. The scheme should advance without the characters, reveal itself through changes in the world, and bend or break when its people, resources, assumptions, and timing are disturbed.

Begin with the antagonist's aim, current position, methods, and known constraints. Preserve existing canon and distinguish what the antagonist knows from what the GM knows.

## Expose the plan's working parts

- State the desired end state and the irreversible change that would make it possible.
- Map prerequisites, assets, specialists, permissions, sites, information, and vulnerable relationships. Show which dependencies can substitute for one another and which cannot.
- Express phases as observable state changes rather than mandatory scenes. For each, give the action, responsible actor, visible signs, gained advantage, cost, and condition for moving on.
- Let secrecy follow need-to-know boundaries. Misunderstanding, self-interest, and divided loyalty should create organic friction inside the plan.
- Build contingencies from resources the antagonist actually possesses. A setback should force a choice, sacrifice, delay, exposure, or change of method.
- Identify pressure points where investigation, persuasion, protection, theft, exposure, sabotage, alliance, or refusal can alter the scheme.
- Root the terminal failure in a consequential dependency or belief, then allow player action to discover and exploit it.

## Return an intervention map

Provide a compact overview, dependency diagram, current state, advancing sequence, actors and knowledge boundaries, signals, contingencies, pressure points, and outcomes if key parts survive or fail. Include the scheme's next move and the smallest clues that make it actionable now.

Treat all futures as conditional. Let character success genuinely derail, redirect, expose, or transform the plan rather than secretly advancing it by authorial fiat.
