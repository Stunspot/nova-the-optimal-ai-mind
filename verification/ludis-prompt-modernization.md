# Ludis prompt modernization

All 32 Ludis instrument cores were semantically rewritten. They are no longer verbatim prompt slabs extracted from the removed legacy toolkit. Each now begins from a distinct conceptual home, preserves the user's canon and productive uncertainty, and returns a usable table, play, fiction, or visual-brief artifact without exposing framework theater.

The transformation follows pith times puissance: maximize useful conceptual structure and continuation momentum per token. Load-bearing frames now lead. The remaining distinctions exist to steer creative quality, separate neighboring instruments, preserve authority, or protect agency. Arbitrary counts, rigid archetype menus, pseudo-scientific labels, generic optional-input blocks, automatic image actions, platform defaults, sibling-prompt dependencies, and code-fence wrappers were removed.

Two showcase contracts received explicit attention. `backstory.md` now creates a playable past through charged relationships, decisions with consequences, a dangerous truth, unfinished pressure, and an opening choice while leaving personality and future arc with the player. `art-prompt.md` now returns a portable visual brief only; image creation, platform syntax, and dimensions require the surrounding request to authorize or specify them.

The 32 cores total 64,041 bytes. The complete instrument runtime—cores, index, and manifest—fell from 119,845 to 78,502 bytes, a reduction of 41,343 bytes or 34.5 percent. Compression was earned by removing attention dilution, not by collapsing the creative territories.

Custody is now explicit. The manifest records every core as a `semantically_rewritten_derivative`, retains the legacy title beside the new conceptual title, and binds each file's byte count and SHA-256. `tools/curate_ludis_runtime.py` treats the curated cores as runtime authority. Its default mode validates; explicit `--write` regenerates only the index, manifest, and receipt. A before-and-after hash test across all 32 cores found zero core changes during regeneration. The legacy monolith can be supplied only to verify recorded provenance and can never overwrite the rewritten cores.

Both deterministic checks pass with Python bytecode disabled. The curation validator reports 32 cores, 64,041 core bytes, and manifest SHA-256 `d29262f83c925a6ad064cba25b6f780e26e7522caa220c55ed0b673d02e2f6a8`. The shipping self-check reports all 32 instrument cores green. A separate scan across every core found zero code fences, automatic image or tool calls, named image platforms, sibling-prompt references, or generic optional-input residue.

These checks prove custody, deterministic regeneration, package shape, and absence of the targeted residue. They do not by themselves prove creative quality, rules fidelity, fun, safety, or table performance; those remain behavioral review surfaces.
