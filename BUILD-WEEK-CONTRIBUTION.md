# What changed during OpenAI Build Week

Nova the Optimal AI + MIND builds on Collaborative Dynamics skills that existed before the contest. The contest project is the product-level system created and materially extended with GPT-5.6 in Codex during the submission period. Judges should evaluate the work below as the Build Week contribution, not assume every source doctrine began this week.

## Prior work used as source material

Collaborative Dynamics entered Build Week with working versions of MIND's cognitive Faculties and several specialist skills now curated into Nova, including TestForge, Agentic Coding, Ludis Continuum, research, retrieval, knowledge stewardship, visual-story work, explicit pins, and governed people context. Those sources were not copied blindly. Their selected roots, imported bytes, exclusions, repository or archive custody, and contest transformations are recorded in `verification/source-import-ledger.json`, `verification/transformation-map.json`, and `verification/nova-skill-reconciliation.json`.

## Material Build Week extension

The contest work created Nova's new front-counter product: one accountable operative identity, direct-first behavior, backstage capability routing, bounded memory explanations, graceful operation without MIND, a conversational first-success tour, a branded offline interactive companion, and a professional first-party Collaborative Dynamics and stunspot dossier with claim-level source custody.

It also turned MIND into a separately installable 1.0.0 Codex plugin, repaired Cognitive Continuity's date-only timestamp regression, rebuilt MIND's integrated capability fingerprint, and validated all sixteen handles as one independently installable product.

The Nova plugin was assembled as a second 1.0.0 product with twelve reconciled skills. Agentic Coding was recast as live repository proprioception rather than a code generator. TestForge became the verification backplane, with an operator that builds evidence and a separate reviewer that attacks it. Ludis retained its high-delight game and fiction capability while every worked setting was removed and its reusable creative instruments were rewritten for compact conceptual salience rather than legacy prompt bureaucracy.

The delivery path was built during the contest as well: a local two-plugin marketplace, deterministic archives, exact source-to-cache comparison, an initially empty host installation probe, source and transformation custody, contamination scans, official plugin and skill validation, TestForge review materials, responsive browser checks, judge instructions, submission copy, and a bounded video-production workflow tested separately on synthetic inputs.

## How GPT-5.6 and Codex contributed

GPT-5.6 in Codex served as the primary product-building environment. It inspected the live source estate; traced provenance and exclusions; designed the two-plugin architecture; edited prompts, code, tests, documentation, branding, and onboarding; found and repaired regressions and contamination; built deterministic verification and packaging tools; exercised installation and browser behavior; and repeatedly challenged the candidate through separate adversarial reviews.

Human judgment remained responsible for product intent, source ownership, public claims, legal and contest attestations, publication, and final acceptance. TestForge kept files, executed behavior, installation, release readiness, and external submission events as separate evidence states.

## Evidence and cutoff

The release hashes, current-host and clean-host receipts, prompt-surface audit, and Nova-only skill reconciliation are bound in the verification package as they are earned. The final Git revision, Build Week task identifier, and required `/feedback` identifier will be added to the submission record only after those events occur. A timestamped file or generated report is evidence only for the claim it actually observes; it is not treated as proof that an external repository, video, or Devpost submission exists.
