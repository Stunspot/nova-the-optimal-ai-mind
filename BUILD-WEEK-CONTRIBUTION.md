# What changed during OpenAI Build Week

During Build Week, GPT-5.6 in Codex turned Collaborative Dynamics' existing cognitive work into two installable 1.0.0 products with one coherent user journey: Nova the Optimal AI and MIND.

## The Build Week contribution

Nova became the accountable front door. The contest work created her direct-first behavior, backstage capability routing, graceful operation without MIND, bounded memory explanations, conversational first-success tour, branded offline companion, and first-party professional context about Collaborative Dynamics and stunspot.

MIND became a separately installable Codex plugin: one integrator plus fifteen Faculties. The work repaired Cognitive Continuity's date-only timestamp regression, rebuilt MIND's integrated capability fingerprint, and validated all sixteen handles as one product rather than a loose collection.

Nova's second plugin package reconciled twelve skills around the front-door experience. Agentic Coding was recast as live repository proprioception rather than generic code generation. TestForge became the release backplane, with one skill building evidence and a separate reviewer attacking it. Ludis kept its game and fiction power while every worked setting was removed; its reusable creative instruments were rewritten around compact concepts and salience.

The delivery system was also built during the contest: a local two-plugin marketplace, deterministic archives, exact source-to-cache comparison, clean installation probes, provenance and transformation ledgers, contamination scans, official plugin and skill validation, adversarial TestForge reviews, responsive browser checks, judge documentation, public submission copy, GitHub publication custody, and a bounded video-production pipeline.

## What existed before the contest

Collaborative Dynamics entered Build Week with working versions of MIND's cognitive Faculties and several specialist skills now curated into Nova, including TestForge, Agentic Coding, Ludis Continuum, research, retrieval, knowledge stewardship, visual-story work, explicit pins, and governed people context.

Those sources were not copied blindly or claimed as brand-new. Selected roots, imported bytes, exclusions, repository or archive custody, and contest transformations are recorded in [`verification/source-import-ledger.json`](verification/source-import-ledger.json), [`verification/transformation-map.json`](verification/transformation-map.json), and [`verification/nova-skill-reconciliation.json`](verification/nova-skill-reconciliation.json).

## How GPT-5.6 and Codex contributed

GPT-5.6 performed the reasoning work in Codex: inspecting the live source estate; tracing provenance and exclusions; designing the two-product architecture; rewriting model-facing surfaces; finding defects, drift, and contamination; composing the onboarding and public story; and challenging the evidence through repeated adversarial review.

Codex supplied the operational surface: repository navigation and editing, local tool execution, installation and browser tests, deterministic packaging, Git custody, verification receipts, and coordination of the contest artifact pipeline.

Human judgment retained product intent, source ownership, legal and contest attestations, final viewing, publication, and acceptance. TestForge kept files, executed behavior, release judgment, and external submission events as separate evidence states.

## Evidence boundary

The release hashes, installation receipts, prompt-surface audit, Nova skill reconciliation, live scenario receipts, and public-repository observation are bound in [`verification/`](verification/) as they are earned. A timestamped file proves only the event it observed. It does not magically prove a public upload, rights attestation, feedback submission, or Devpost status.
